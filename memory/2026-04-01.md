## MLS Lead Scraper - Fixed and Rebuilt

### Problems Found
- Original scraper used wrong login URL (hardcoded SAML token that expired)
- Login form field selectors were wrong (`name="username"` → needs `name="member_login_id"`)
- Login form has Email/MLS Username radio toggle — must click Email first
- `By.TYPE` isn't a valid Selenium locator (submit button)
- Post-login wait was looking for wrong elements (`.dashboard`, `.search`, `#main-content`)
- **Biggest issue:** Original scraper was trying to search within SolidEarth portal — the actual MLS search is in **Paragon** at `ccar.paragonrels.com`

### Architecture Discovered
- **SolidEarth** (`ccar.mysolidearth.com`) = SSO/identity portal only
- **Paragon 5** (`ccar.paragonrels.com`) = actual MLS system with listings/search
- SSO login carries auth cookies to Paragon automatically
- Paragon uses iframes heavily: `HomeTab` iframe has Market Monitor and Quick Search
- Market Monitor has direct links: Expired (142), Price Change (186+), Back on Market, etc.
- Clicking those opens results in new tab iframes (`tab1_1`, etc.)
- Results have Export menu → Export to CSV → dialog with Export button

### New Scraper Flow
1. Login to SolidEarth SSO (Email radio → email + password → submit)
2. Navigate to `http://ccar.paragonrels.com/`
3. Close board message overlay
4. Switch to HomeTab iframe
5. Click Market Monitor category links (Expired, Price Change)
6. Switch to results tab iframe
7. Export menu → CSV → click Export button in dialog
8. Parse downloaded CSV, filter by target areas
9. Save dated report CSV
